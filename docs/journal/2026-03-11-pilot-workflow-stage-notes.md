# Pilot Workflow Definition Stage Notes - 2026-03-11

## 1. 为什么现在必须收敛到一条 workflow

平台如果一直停留在“支持很多未来可能性”，就很难真正进入企业现场。制造业不缺概念，缺的是一条可以被业务负责人、质量负责人、设备负责人共同接受的真实落地路径。

## 2. 这一阶段的创新点

这一阶段的创新，不在于再加一个新技术名词，而在于把 Agentic AI 平台从“平台能力叙事”推进到“具体业务闭环设计”。

这一步要求我们回答非常具体的问题：

- 哪条流程最值得先做
- 哪条流程风险最低但价值足够高
- 哪些动作可以由 AI 提建议，哪些动作必须保持人工批准
- 哪些系统读取是必要的，哪些写操作必须继续禁止

## 3. 这如何改变世界

真正改变制造现场的，不是一个能回答很多问题的模型，而是一条能稳定进入 SOP、被责任人接受、能持续复用的工作流。

第一条 workflow 一旦选对，后面平台的连接器、证据模型、审批模型、试运行验收都会开始围绕真实价值收敛。这比继续增加抽象层更重要。

## 4. 对自己的要求

- 不为了显得先进而选一个超范围 workflow
- 不把“生成建议”偷换成“自动执行”
- 所有创新都必须能落到角色、证据、审批、回退这四件事上
