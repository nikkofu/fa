# Evidence Stage Notes - 2026-03-12

## 1. 为什么这一步重要

Agentic AI 在制造场景里最容易失去信任的地方，不是模型答错一句话，而是系统给了建议，却说不清证据从哪里来、为什么要这么建议。

如果 evidence 只是散落在 connector 返回里，平台就还没有真正具备“证据驱动”的形态。

## 2. 这一阶段的创新点

这一阶段不是去做更大的模型，而是把“证据”从隐式数据推进成显式系统对象。

这意味着：

- 任务不仅有状态，还有证据快照
- API 不只是返回动作结果，也返回建议背后的证据基础
- 审计回放之外，任务本身开始具备更强的可解释性

## 3. 这如何改变世界

真正能进入制造现场的 AI 平台，必须让一线负责人看到的不只是“系统建议你这样做”，而是“系统为什么这样建议，它看到了哪些事实”。

证据对象化是让平台从“会编排”走向“可被追责、可被信任”的关键一步。

## 4. 对自己的要求

- 先做轻量、真实可用的 evidence snapshot
- 不提前发散到大而全的 knowledge platform
- 每个新增对象都必须同时服务 API、持久化、审批理解和试运行演示

## 5. 已经验证的事实

- 任务在创建时会生成结构化 evidence 快照
- evidence 会跟随任务状态一起持久化和回读
- evidence 既能在任务详情里看到，也能通过单独接口查询

这说明平台开始具备“把证据作为正式交付对象”的能力，而不只是“内部用过一些上下文数据”。

## 6. 这次做对了什么

这次做对的地方，是没有把 evidence 设计成一个过早膨胀的大系统，而是让它先贴着当前任务主链成长。

这样做的好处很直接：

- API 立刻可用
- 文件和 SQLite 持久化天然继承
- 审批和试运行演示可以马上消费这层信息

## 7. 这一步如何真正产生影响

制造场景里的信任，不来自“模型很强”，而来自“系统能把依据讲清楚”。当 evidence 进入正式输出，平台就更接近一个可被现场负责人接受的工作系统，而不是一个只能展示生成能力的工具。
