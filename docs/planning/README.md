# Planning Docs

本目录用于存放版本级、里程碑级和阶段级执行计划，目的是把路线图转成团队可执行、可跟踪、可验收的工作包。

## 使用规则

- `roadmap.md` 定义方向和版本目标。
- `planning/` 负责把目标拆解成 WBS、责任、依赖、验收和节奏。
- 每个进入执行的里程碑或版本，至少需要一份执行计划文档。
- 执行计划中的工作项，后续应映射到 GitHub issue / project board。
