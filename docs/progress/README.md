# Progress Docs

本目录用于记录项目每个阶段开始前和阶段推进中的正式进展同步。

## 目的

在进入新的阶段任务前，强制把以下信息写回仓库，确保项目状态不是口头的，而是可追溯的：

- 当前版本状态
- 当前阶段目标与范围
- 已完成项与未完成项
- GitHub 仓库同步状态
- 风险与下一步
- 工作心得与项目影响说明

## 使用规则

每次进入新的阶段任务前，至少同步以下内容：

1. 更新 `CHANGELOG.md`
2. 更新相关 `roadmap / planning / README`
3. 新增或更新一份阶段进展文档
4. 记录 GitHub 远端同步状态
5. 记录本阶段的工作心得和项目影响说明
