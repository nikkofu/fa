# Local Development Environment

## 1. 目的

本文件定义 FA 项目的本地开发端口和基础运行约定，避免与同机上的其他项目产生端口冲突。

## 2. 默认端口约定

- `fa-server` 默认监听 `8000`
- 默认环境变量：`FA_SERVER_ADDR=0.0.0.0:8000`

选择 `8000` 的原因：

- 与本机上常见的 `8080`、`3000`、`5173`、`38080` 类项目端口做出隔离
- 降低与其他 API server、前端 dev server、历史测试服务的默认冲突概率

## 3. 本地冲突规避规则

- 不直接修改代码中的默认端口来适配个人环境
- 个人机器如存在冲突，优先通过环境变量覆盖
- README、脚本和示例命令默认以 `8000` 为基准
- 新增本地服务时，先检查是否会与 `8000` 冲突

## 4. 覆盖方式

示例：

```bash
FA_SERVER_ADDR=127.0.0.1:8010 cargo run -p fa-server
```

或在本地 `.env.local` 中设置：

```text
FA_SERVER_ADDR=127.0.0.1:8010
```

说明：

- 团队共享文档、脚本、示例请求仍以 `8000` 为默认值
- 个体开发者的临时覆盖不应反向提交到仓库

## 5. 本地持久化模式

如果需要在本地重启后仍保留任务和审计数据，可设置：

```bash
FA_DATA_DIR=/tmp/fa-data cargo run -p fa-server
```

说明：

- 未设置 `FA_DATA_DIR` 时，服务继续使用内存模式
- 设置 `FA_DATA_DIR` 后，任务状态会写入 `tasks/*.json`
- 审计事件会写入 `audit-events.jsonl`
- 该模式用于本地试运行和验证，不等同于数据库级持久化

## 6. 推荐本地检查方式

启动前先检查端口是否被占用：

```bash
lsof -iTCP:8000 -sTCP:LISTEN
```

如果有占用：

1. 确认是否为当前项目历史进程
2. 若不是当前项目，使用 `FA_SERVER_ADDR` 切换到其他端口
3. 不要为了临时运行而改仓库默认值

## 7. 后续约定

后续如果增加前端、worker、mock connector server 或本地依赖服务，应继续在本目录下补充统一端口表，避免项目规模扩大后再次发生端口混乱。
